Component Life Cycle Example

  • Simple app displays "true" text and then on clicking, it changes it to display "false"
  • View the console -there is a simple console.log message for each lifecycle method
  • The order of run
    1. getDefaultProps
    2. getInitialState
    3. ReactDOM.render triggerscomponentWillMount Entry point for our component. Method called when a component gets mounted first time.
      componentWillMount is where you can call setState to change peform changes to our internal data (if required). However, this does not call a new re-render or this method
    4. The actual render method then executes, and is responsible for the component display
    5. componentDidMount invoked immediately after mounting and rendering of the component
      This method can be used to fetch dynamis information we want to display in the component
    6. Thre is a simple onClick event. This calls this.toggleState, which toggles the current status from true to false and vice versa.
      As state is affected, React re-renders the App component - via the same render function
      When state is changed, React knows it needs to re-render
    7. React calls shouldComponentUpdate first. This method returns true or false indicating if React renders
      If this returns false, the render function is not called
      You can compare nextProps and nextState to the existing values and then decide whether to re-render
      By default this always returns true
    8. componentWillUpdate called just before rendering
      You can under any last minute changes / housekeepping Of Note: You can not call setState in this method
    9. componentWillReceivePropsreceives nextProps, which are the new props that the child component receives from the parent component.
      This method is not called for initial render. We can update the state or do some other housekeeping work based on the changes in props.
    10. componentWillUnmount is called when the component is unmounted, this is normally used to release the resources, perform cleanups, unset any timers, and so on.
